# Nested Stack, Tab, and Drawer Navigation

Since we can build apps with the different types of Navigators, and we can nest different types inside of other types, the following repo is an example of how all three could be combined.

Repo for Nested Navigation (opens new window)

The example app is set up as follows:

  • Tab navigator creates the tabs at the bottom that links to Home, About, and Contact.
  • Inside of About there is a Stack navigator that lets us toggle between a This and That page.
  • Inside the Contact page there is a Drawer navigator with a link back to the Contact page plus a link to an external URL, which will open the browser to load the URL.

# Expo Router

The alternative to using React Navigation is to use Expo Router. Expo Router is built as a wrapper around React Navigation and it provides file-based routing.

Expo Router reference (opens new window)

First, let's start with a blank project.

npx expo create-expo-app@latest . --template blank
1

To install it in your React Native app you can run the following. The command will add support for a number of other related modules.

npx expo install expo-router react-native-safe-area-context react-native-screens expo-linking expo-constants expo-status-bar
1

Once the project has been created and the dependencies installed then we need to create an app folder and update the starting files. We will set up a basic _layout.js file and an index.js file. These will be the start of the application. You can use .js, .jsx, .ts, or .tsx as the file extensions, just be consistent.

You may optionally wrap all your app folders and files, include the app folder, inside of a src folder.

Next we need to point at those files as the entry point for the application. We do this in the package.json file by updating the "main" property to be "expo-router/entry". This will automatically point to the _layout.js file inside of the app folder.

For proper routing and deep linking, we also need to open the app.json and add the scheme value. It can be https://, http://, or a special scheme that you have defined for your app like myapp://. Unless you have a specific reason, https:// should be the value that you use.

{
  "expo": {
    "scheme": "https://"
  }
}

The key difference between React Navigation and the Expo Router approach, is that Expo Router is file based. As we create folders and files inside of /app, each becomes a new route (except for _layout.js). Then we can start creating app routes by naming the files inside the app folder. Each of the following creates a route.

  • app/index.jsx is route /
  • app/products.jsx is route /products
  • app/contact.jsx is route /contact
  • app/stuff/things.jsx is route /stuff/things

index.jsx will be the home page for the current folder. If you create additional folders inside of /app/ then each folder should get an index.jsx file. We can use .js, .jsx, .ts, or .tsx as valid file extensions.

If you use _layout.jsx inside any folder then it becomes the layout file used as a wrapper for every route inside that folder.

If you want a file to be able to handle any other value, then wrap the name with square brackets. An example would be app/[id].jsx or app/[id]/index.jsx. This file would be the route for any other routes inside of app, where there is no matching filename. The value inside the square brackets, known as a slug or route param, can be used as a variable.

Routes that use index as the filename just use the folder name as the route. They are the default file for that folder. This is just like index.html as the default file for a folder in a website.

If you want to access the value of the dynamic part of the route, the param, then import useLocalSearchParams hook from expo-router.

import { useLocalSearchParams } from 'expo-router';

export default function Page() {
  const { slug } = useLocalSearchParams();
}

If you want to build reusable components, then do not put those files inside the app folder, create your components folder outside of the app folder.

# Navigating Between Routes

To navigate between pages, import and use the <Link> component with an href attribute with the route.

import { View } from 'react-native';
import { Link } from 'expo-router';

export default function Page() {
  return (
    <View>
      <Link href="/about">About</Link>
      {/* ...other links */}
      <Link href="/user/bacon">View user</Link>
    </View>
  );
}

# Expo Router Hooks

usePathname() - returns the path part of the url without querystring

useNavigation() - provides the navigation object for doing navigation, like navigation.openDrawer().

useRouter() - provides the router object for doing navigation between routes, like router.push('/home').

useLocalSearchParams() - provides an array of params in the querystring like const { user, extra } = useLocalSearchParams();. Similar to the URLSearchParams.

When we create a _layout.jsx file as the layout container for a folder, we define the location to place the content from the other files by importing and using a <Slot> component. import { Slot } from 'expo-router';

# Tabs and Stacks

At the top level, inside /app/_layout.tsx we want to add a <Stack> as a wrapper for the navigation. Here are the minimal contents of /app/_layout.tsx.

import { Stack } from 'expo-router';

export default function RootLayout() {
  return <Stack />;
}

With this in place you can create your screen files inside the app folder and then add <Link href=""> or <Button onPress={}> components on each page to construct the navigation.

However, if you want a tab bar at the bottom of the screen then we are going to create a folder inside of app called (tab). Be sure to include the parentheses around the word tab.

We will place our screen files inside the (tab) folder, and then update the /app/_layout.tsx file to be this:

import { Stack } from 'expo-router';

export default function RootLayout() {
  return (
    <Stack>
      <Stack.Screen name="(tabs)" options={{ headerShown: false }} />
    </Stack>
  );
}

It is pretty much the same but we add a <Stack.Screen> whose name property loads the (tabs) folder.

Inside the (tabs) folder, we create another _layout.tsx file that defines the tabs.

import { Tabs } from 'expo-router';
import FontAwesome from '@expo/vector-icons/FontAwesome';

export default function TabLayout() {
  return (
    <Tabs screenOptions={{ headerShown: false }}>
      <Tabs.Screen name="index" options={{ title: 'Home', tabBarIcon: ({ color }) => <FontAwesome size={28} name="home" color={color} /> }} />
      <Tabs.Screen name="two" options={{ title: 'Stuff', tabBarIcon: ({ color }) => <FontAwesome size={28} name="user" color={color} /> }} />
      <Tabs.Screen name="three" options={{ title: 'Settings', tabBarIcon: ({ color }) => <FontAwesome size={28} name="cog" color={color} /> }} />
    </Tabs>
  );
}

# Redirects

Sometimes you will need to check some state value and then redirect the user to a different route if the value is not what you wanted. The Redirect component can be used to send the user to a different route, like the following.

import { View, Text } from 'react-native';
import { Redirect } from 'expo-router';

export default function Page() {
  const { user } = useAuth(); //fake custom hook

  if (!user) {
    return <Redirect href="/login" />;
  }

  return (
    <View>
      <Text>Welcome Back!</Text>
    </View>
  );
}

# Vector Icons

Expo comes with a component that you can use to add icons to any app. The Expo Vector Icons (opens new window) component can be used with any project that was created with expo-create-app. It is installed by default.

To add it to a page add the import for the family of icons that you want to use.

import Ionicons from '@expo/vector-icons/Ionicons';
1

The base path is @expo/vector-icons/ and then you add the family at the end of the path plus the name of the component that you are importing. This will give you access to all the fonts in that family.

List of Icons (opens new window) - you can search for icons on this page.

The icon families include: Ionicons, FontAwesome, FontAwesome5, Entypo, AntDesign, Zocial, SimpleLineIcons, Octicons, EvilIcons, Feather, Fontisto, Foundation, MaterialIcons, and MaterialCommunityIcons.

Once you have imported the set that you want, then you can use the component that you imported in your JSX. Add a name, size, and color attribute to display the icon.

<Ionicons name="md-checkmark-circle" size={32} color="green" />
1

You can do this as a child component inside a <View>, a <Button>, a <Pressable>, or most other components.

# Platform Specific Code

If you are looking to change things in your app based on which OS the user is currently using, then you can do that with the Platform module. It is a built-in React Native module. No install required.

import { Platform } from 'react-native';
1

The Platform object is not a component to use in your JSX. It is an object that you can use in your JavaScript when deciding what to add to your JSX or its properties.

The Platform object has a lot of different properties that you can access through Platform.constants. but the one property that will be most useful is Platform.OS which will have a value of web, ios, or android.

You can use the value of Platform.OS to change how your component renders.

export default MyComponent()=>{
  let os = Platform.OS;
  let mystyle = {};
  if(Platform.OS === 'ios'){
    mystyle.backgroundColor = 'red';
  }else if(Platform.OS === 'android'){
    mystyle.backgroundColor = 'blue';
  }else{
    mystyle.backgroundColor = 'green';
  }
  return (
    <View style={mystyle}>
        <Text>{os}</Text>
    </View>
  );
}

# Floating Action Button (FAB)

A FAB is a type of button that you will see in a lot of Android apps plus some iOS apps. It is a common design pattern in Material Design.

There is no built-in FAB in React Native or Expo.

You can build your own custom one with the <Pressable> component as the root. Here is a sample:

export FAB (props) => {
    return (
        <Pressable style={styles.container}
            onPress={props.onPress}>
            <Text style={styles.title}>{props.title}</Text>
        </Pressable>
    );
};


const styles = StyleSheet.create({
    container: {
        /* position the content inside the FAB */
        justifyContent: 'center',
        alignItems: 'center',
        paddingHorizontal: 20,
        paddingVertical: 10,
        /* for the FAB itself */
        borderRadius: 10,
        position: 'absolute',
        bottom: 70,
        right: 40,
        backgroundColor: '#26653A',
    },
    title: {
        /* style the text inside the FAB */
        fontSize: 18,
        color: '#fff',
        fontWeight: 'bold',
    },
});

# PlatformColor

In your styling for your apps you can use any colour that you would typically use in your CSS.

In your JS code for your React Native app, you can also use the PlatformColor function to access native colors on the target platform by supplying the native colors corresponding string value. React Native PlatformColor (opens new window)

You pass a string to the PlatformColor function and, provided it exists on that platform, it will return the corresponding native color, which you can apply in any part of your application. The first colour is the default and the others are fallbacks.

PlatformColor(color1, [color2, ...colorN]);
1

These are colours that you would define within the Android and iOS native settings.
