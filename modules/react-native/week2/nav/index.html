# Nested Navigation

The <Stack.Navigator> is the first navigator that we discussed last week. There are other Navigators too - TabNavigator and DrawerNavigator. We will discuss both of those on this page and in class.

It is possible to nest one inside another. We can have a Stack navigator with a different Drawer navigator inside each route. We can have a Tab navigator with a different Stack navigator used inside each of the Tab routes. Or any other combination you want.

The options for the the various screens can change a bit when you nest navigators. Screen options with nested navigators reference (opens new window)

# Bottom Tab Navigation

The usage instructions for Tab.Navigator are very similar to the Stack.Navigator. If you have built even one simple app using the <Stack.Navigator> and <Stack.Screen> components, then you should have no problem quickly learning the Tab version.

Create your starter app first.

npx create-expo-app@latest . --template blank
1

Before we can use the tab navigator, we need to install the appropriate NPM module. See the reference for the Bottom Tab Navigator (opens new window). After creating your project we will want the same install commands as we did previously with the Stack, but add the one for bottom-tabs. The bottom-tabs is the last step in the following installs

npm install @react-navigation/native

npx expo install react-native-screens react-native-safe-area-context

npm install @react-navigation/native-stack

npm install @react-navigation/elements

npm install @react-navigation/bottom-tabs

Just like the Stack version, we need a NavigationContainer at the top level of our app.

Next create a new instance of the Tab object using the imported function. When invoked, it returns on object with two properties containing React components: Screen and Navigator.

const Tab = createBottomTabNavigator();
1

Then in the JSX, replace Stack.Navigator with Tab.Navigator and replace Stack.Screen with Tab.Screen.

export default function App() {
  return (
    <NavigationContainer>
      <Tab.Navigator
        initialRouteName="Home"
        options={{
          tabBarLabelPosition: 'below-icon',
          tabBarLabelStyle: {
            fontSize: 16,
          },
          tabBarIconStyle: {
            //style for the icon on every tab
            color: 'steelblue',
          },
        }}
      >
        <Tab.Screen
          name="Home"
          component={HomeScreen}
          options={{
            title: 'Home',
            tabBarBadge: 3, //add a badge to the icon
            tabBarIcon: ({ focused, color, size }) => {
              //focused is a boolean that we can use to change the icon or color
              return <MaterialIcons name="home" size={size} color={color} focused={focused} />;
            },
          }}
        />
        <Tab.Screen
          name="About"
          component={AboutScreen}
          options={{
            title: 'About',
            tabBarBadge: 3, //add a badge to the icon
            tabBarIcon: ({ focused, color, size }) => {
              //focused is a boolean that we can use to change the icon or color
              return <MaterialIcons name="display-settings" size={size} color={color} focused={focused} />;
            },
          }}
        />
        <Tab.Screen
          name="Contact"
          component={ContactScreen}
          options={{
            title: 'Contact',
            tabBarBadge: 3, //add a badge to the icon
            tabBarIcon: ({ focused, color, size }) => {
              //focused is a boolean that we can use to change the icon or color
              return <MaterialIcons name="alternate-email" size={size} color={color} focused={focused} />;
            },
          }}
        />
      </Tab.Navigator>
    </NavigationContainer>
  );
}

# Add Icons

It is easy to add icons to your tabs using the included react-native-vector-icons packages that Expo installs by default. You can search through all the available icons here (opens new window).

First, import the family of icons that you want. Here are a couple examples.

import Ionicons from '@expo/vector-icons/Ionicons';
import MaterialIcons from '@expo/vector-icons/MaterialIcons';
1
2

Then, if you want to use any icon from either of those families on your screen, the imported component is the icon rendering component. The name is the name of the icon within the family, the size is density independent pixels just like the fontSize, and the color, is any valid color.

<Ionicons name="umbrella" size={24} color="black"/>
<MaterialIcons name="nightlight" size={24} color="black" />
1
2

With the <Tab.Screen> components, using the options prop, set the tabBarIcon. To make the icon dynamic, use a function that is given the focused state, color, and size params.




 
 
 
 
 


<Tab.Screen
  name="Contact"
  component={ContactScreen}
  options={{
    tabBarIcon: ({ focused, size, color }) => (
      <Ionicons name="ios-paper-plane" size={size} color={color} />
    ),
  }}
/>

# Drawer Navigation

A Drawer Navigator is a panel that the user can drag out from the side of the screen. We can open and close the Drawer with a script command from any function, or we can let the user drag it out themselves. There will be a <Drawer.Navigator> which has a series of <Drawer.Screen> components.

The full reference for the Drawer Navigator is here (opens new window).

Install the drawer module along with the other navigation requirements, like before. If we are going to add Tab navigation as well, we should install the @react-navigation/bottom-tabs module too. So, we will have the Stack, Tab, and Drawer imports, plus we need to install the expo gesture-handler and reanimated packages.

It would be very rare to only have the Drawer Navigator without either tabs or stack.

npm install @react-navigation/native @react-navigation/native-stack
npm install @react-navigation/bottom-tabs
npm install @react-navigation/drawer

npx expo install react-native-screens react-native-safe-area-context
npx expo install react-native-gesture-handler
npx expo install react-native-reanimated

The Gesture Handler needs to be imported as the first line in your App.js file. Note that we are not importing a specific method or property, just the file. The code in that file needs to be run so the gesture handling events are made available to us.

The reanimated import is for the drawer to be animated on and off the screen following the users' gestures.

The imports need to include the direct loading of the react-native-gesture-handler as the first import on the root file of your project.

import 'react-native-gesture-handler'; //must be the first import on index.js

//Then the following are in App.js
import { NavigationContainer } from '@react-navigation/native';
import { createDrawerNavigator, DrawerContentScrollView, DrawerItemList, DrawerItem } from '@react-navigation/drawer';

const Drawer = createDrawerNavigator();

The App.js file sets up the basic top level navigation structure for the App. Each Drawer.Screen component will have an entry in the Drawer that slides out from the side of the screen. Just like the Tab and Screen navigators, there needs to be a <NavigationContainer> at the top level of the app, as a Provider.

There will be a <Drawer.Navigator> around the one or more <Drawer.Screen> components. The navigator will have a drawerContent property that holds a function which builds the content inside of the Drawer that slides out from the side.

The drawerContent should start with a <DrawerContentScrollView> component, imported from @react-navigation/drawer.

Nested inside of the <Drawer.Navigator> will be one or more <Drawer.Screen> components. Each of these can have either a component attribute, or a nested function which creates the content. The example below shows a single nested function building the component inside the screen with the name InContact. The route Contact is this screen, which contains the navigator. The route InContact represents the content inside of the <Drawer.Screen>.

export default function App() {
  return (
    <Drawer.Navigator initialRouteName="InContact" screenOptions={{ headerShown: false }} drawerContent={(props) => <CustomDrawerContent {...props} />}>
      <Drawer.Screen name="InContact">
        {(props) => (
          <View style={styles.container}>
            <Text style={styles.title}>Contact Screen</Text>
            <Text style={styles.title}>{id}</Text>
            {/* Add the Navigation Buttons  */}
            <Button style={styles.btn} onPress={() => goto('Home')}>
              Go To Home
            </Button>
            <Button style={styles.btn} onPress={() => goto('About')}>
              Go To About
            </Button>
            <StatusBar style="auto" />
          </View>
        )}
      </Drawer.Screen>
    </Drawer.Navigator>
  );
}

Just like with the Tab navigator you can use the default drawer and the screens will automatically added as items in the list inside the drawer. The <Drawer.Screen> components are the props.children being passed to the <CustomDrawerContent>.

Our own custom container for the drawerContent prop in the <Drawer.Navigator> would be something like the following:

function CustomDrawerContent(props) {
  return (
    <DrawerContentScrollView {...props}>
      <DrawerItemList {...props} />
      <DrawerItem label="Help" onPress={() => Linking.openURL('https://reactnative.dev/')} />
    </DrawerContentScrollView>
  );
}

The DrawerContentScrollView, DrawerItemList, and DrawerItem must all be imported.

Inside the DrawerContentScrollView you will have at least a DrawerItemList. With the props being passed in, there will be a DrawerItem built for each of the <Drawer.Screen> items.

If we want to add additional items, or even only add custom items then we can just use <DrawerItem> components.

In the example, the onPress event calls on Linking.openURL(url), which is a method imported from react-native. This built-in method will open a URL in the device default browser.

However, remember that the name attributes added to all of your app's <___.Screen> components must have unique values.

# Refresh Control

When you want your user to be able to drag down from the top of the screen with their finger and have that gesture trigger code that refreshes some of your content, use the RefreshControl.

RefreshControl built-in RN component (opens new window)

import { useState, useCallback } from 'react';
import { RefreshControl, SafeAreaView, ScrollView, StyleSheet, Text } from 'react-native';

export default () => {
  const [refreshing, setRefreshing] = useState(false);

  const onRefresh = useCallback(() => {
    setRefreshing(true);
    //refreshing used in <ScrollView> refreshControl attr
    //which loads the <RefreshControl> with the refreshing attr
    setTimeout(() => {
      setRefreshing(false);
    }, 2000);
  }, []);

  return (
    <SafeAreaView style={styles.container}>
      <ScrollView contentContainerStyle={styles.scrollView} refreshControl={<RefreshControl refreshing={refreshing} onRefresh={onRefresh} />}>
        <Text>Pull down to see RefreshControl indicator</Text>
      </ScrollView>
    </SafeAreaView>
  );
};

const styles = StyleSheet.create({
  container: {
    flex: 1,
  },
  scrollView: {
    flex: 1,
    backgroundColor: 'pink',
    alignItems: 'center',
    justifyContent: 'center',
  },
});
