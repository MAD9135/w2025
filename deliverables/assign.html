# 1 - React Native App

The React Native assignment is to build an expo powered React Native app that can run on both iOS and Android devices. It will demonstrate your understanding of:

  • React Native navigation between screens.
  • Use of state variables and hooks in React Native.
  • Basic styling of a React Native app.
  • Make API calls from within a React Native app.
  • Use of persistent storage to hold, read, and retrieve data.

Start your project by creating a blank Expo project with:

npx create-expo-app@latest . --template blank
1

The app will have three screens:

  1. A home/welcome screen
  2. A data list screen that uses a FlatList to display data from the Random Data API v2 (opens new window) API. You may use another API for user data like https://randomuser.me (opens new window), as long as it returns different data with each call, and provides enough varied data and you have an image in the data to show on the details screen.
  3. A details screen to show information about one item from the list on the data list screen.

The app will fetch data from the Random Data API and save it locally with either the AsyncStorage or expo Secure Storage package.

Each screen needs to communicate directly with one of the Storage packages. No context API should be used for this assignment.

The navigation in the app can be built using either React Navigation or expo router. The navigation between the home screen and the list screen needs to be managed with bottom tabs. The navigation between the list screen and the details screen will be managed as a Stack within the List tab space.

# Home Screen Details

The home screen will display a welcome message to the user. It will be the default screen shown when the app loads.

This screen should check if there is currently any data saved in your preferred persistent Storage solution (Async or Secure). If there is NO data saved yet, make the fetch call to the random data api and save that data results array from the fetch Response to the storage solution.

The data can be ANY of the Beers, Banks, Users, or Addresses data types from the random data API. Pick ONE.

You can also add a button to the home screen that lets the user clear the list from persistent storage.

# List Screen

The list screen will use a FlatList to display the results from your preferred persistent Storage solution. With a useEffect hook or useFocusEffect hook, check if there is data available in the persistent Storage. If there is an array, then set a state variable to hold that data and display it in the FlatList.

If no data is available in persistent Storage then redirect the user back to the home screen.

The FlatList should have a RefreshControl. If the list is pulled down by the user the spinner will be displayed and the app should make a NEW fetch call to the random data API and save the NEW data in persistent Storage. Then update the state variable being used by the FlatList for showing the data.

Each List item in the FlatList needs a pressable icon that will take the user to the other stack screen - the details screen. The user needs to be able to navigate back from the details to the list screen. The id or unique id from the item needs to be passed to the details screen as a param.

Each list item should display TWO pieces of information from the persistent storage.

# Details Screen

The details screen will show at least 8 pieces of information about the selected item from the list screen. Use the hook to read the id from the param and use the id to find the appropriate matching item from data array.

The app will need to use the useEffect hook and read the data from the persistent storage in order to get the array and then match the id from the params. Remember that the useEffect function itself cannot be async.

If there is no id param or no persistent storage then send the user back to the home screen.

Use a state variable to hold all the details about the one item from the persistent storage data array.

The screen should display at least 8 fields from the matching object in the data array.

# Styling

The app should have some default consistent styling on every screen. The StyleSheet properties object should be created in a single file that gets imported by each screen that needs to style any component. This way the styles will be the same on every page.

Create reusable style objects that can be used on multiple pages.

Do NOT use a UI Kit package as part of this assignment.

# Git Repo

You need to use a Private Github Repo to store your app code. There should be AT LEAST 5 commits in the history. The first commit will be the initial files, right after creating your project and/or doing some imports for navigation and storage.
